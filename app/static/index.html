An investigator's toolkit for the open web.

FalconEye is a free, self-hosted OSINT workbench built for cybersecurity investigators, fraud analysts, and journalists. It collapses the dozen browser tabs an investigator usually juggles — wallet explorers, RDAP lookups, certificate transparency, ASN tools, phishing kit analysis, malware databases, Telegram intelligence — into one focused interface.

Every tab takes a single indicator as input and returns structured intelligence pulled from free, reputable public data sources. There is no paywall, no enterprise sales call, no API budget. Just the tools.

Built and maintained by OSINT-PH — a Philippine-based OSINT and incident response practice. The platform is open source under AGPL-3.0 at github.com/osintph/falconeye.

What this does

  • → Trace crypto wallets across BTC, ETH, and USDT TRC20
  • → Fingerprint phishing kits by their indicators
  • → Profile domains with RDAP, DNS, and CT logs
  • → Check IP reputation, open ports, and scanner activity
  • → Resolve any Telegram user, bot, or channel with tiered intelligence
  • → Look up URLs and file hashes in community threat databases

Domain Intelligence

Paste a domain for unified RDAP, DNS, certificate transparency, and ASN intelligence. Results cached for 6 hours.

Telegram Intelligence

Paste a Telegram username, @handle, channel, or t.me link. Resolves the entity type and extracts all public metadata, messages, and IOCs. Public data only.

Privacy: the identifier is looked up three ways — Telegram's public t.me web preview, the Bot API (channels/groups only), and, if the server operator has authenticated an MTProto session, direct entity resolution. Each tier degrades independently if unavailable; results cached 6 hours.

IP Reputation & Threat Lookup

Paste a public IPv4 or IPv6 address for a multi-source reputation verdict (AbuseIPDB, VirusTotal, AlienVault OTX, Censys, ThreatFox), ASN, geolocation consensus, merged open ports, vulnerabilities, scanner activity, and URLhaus history. Results cached for 6 hours.

Privacy: the IP you look up is sent to external reputation services — AbuseIPDB, VirusTotal, AlienVault OTX, Censys, and ThreatFox, plus Shodan, GreyNoise, RIPEstat, and URLhaus.

File & URL Sandbox History

Paste a URL or a file hash (MD5, SHA1, SHA256) to query URLhaus and MalwareBazaar for community threat intel. Results cached for 6 hours.

URL Expander

Paste a short URL to reveal the full redirect chain, TLS chain, and final landing page. Results are not cached.

Privacy: the URL you submit is fetched by the FalconEye server (not your browser) to follow its redirects; your IP is recorded briefly for rate limiting.

Email Header Analyzer

Paste a complete email header (and optionally the body) to analyze authentication, sender alignment, routing, and scam patterns. You can also upload a .eml or .msg file to skip the copy-paste step.

Privacy: Uploaded files and pasted headers are processed in memory only. The original content is NEVER written to disk and NEVER logged. The parsed analysis result is cached for 24 hours by content hash (so re-analyzing the same email returns instantly), but the raw email itself is discarded immediately after parsing.

Username Enumeration

Check where a username appears across ~965 platforms using WhatsMyName and Sherlock data. Cross-validated hits (found by both engines) carry higher confidence. Results are leads for human verification, not proof of identity.

Privacy: the username you submit is checked against ~965 external platforms (WhatsMyName + Sherlock) by requesting each site directly.
How to use this. A hit means the username has a profile at that platform. It does NOT mean the same person owns all matching profiles. False positives run 5-10% because some platforms return the same response for any username. Verify each lead manually before drawing conclusions. Do not use this tool for stalking or harassment.

Privacy Policy

Effective date: 25 June 2026
Last updated: 20 July 2026
Operator: OSINT-PH (blog.osintph.info)

Summary

FalconEye is a free, self-hosted OSINT investigator's toolkit. This policy explains what data we collect when you use the live instance at falconeye.osintph.info, what we do with it, and who it is shared with.

Short version: input you submit is processed to produce analysis results, cached for 24 hours by content hash, and then discarded. We do not maintain user accounts. We do not sell data. We do not run third-party advertising or analytics. The only third parties that ever see your input are the OSINT data providers we query on your behalf (when applicable) and Anthropic (only for LLM-powered tabs).

What data we collect

  • Input you submit. Crypto addresses, domains, URLs, IPs, email headers, email bodies, file hashes, Telegram channel names, search goals. This is what the tool is for.
  • Uploaded files (Email Header and QR Analyzer tabs). Email .eml / .msg files and QR images are parsed or decoded in server memory and discarded immediately. The original file content is never written to persistent storage and never logged.
  • Source IP address. Used for rate limiting on the rate-limited features (the LLM tabs — Email Header, Dork Generator, Script Decoder — plus URL Expander and QR Analyzer). The IP is read from Cloudflare's CF-Connecting-IP header, stored in our SQLite database with a timestamp, and older rows are automatically deleted.
  • Abuse-report send audit (operator sends only). The Report Abuse cards compose a report you can copy and send from your own email. If the FalconEye operator sends one via Mailgun, an append-only audit row records the timestamp, source IP, recipient abuse-desk address, the reported target and category, the subject line, and the Mailgun message ID — never the report body or evidence.
  • nginx access logs. Standard web server logs: timestamp, source IP, requested URL, response status, user agent. Used for security monitoring and capacity planning. Retained for 30 days, then deleted by log rotation.

What we do NOT collect

  • User accounts. There is no signup, no login, no profile.
  • Tracking cookies. FalconEye sets one functional cookie — fe_theme, which stores your light/dark theme preference in your own browser — and nothing else. It carries no identifier and is never transmitted to any third party. (Cloudflare may set its own security cookies; see the Cloudflare section below.)
  • Browser fingerprints, device IDs, or behavioral analytics.
  • Email addresses or personal identifiers, unless you voluntarily paste them as input.
  • The raw content of uploaded .eml / .msg files or QR images after parsing.
  • The content of PDF reports you generate. IP-reputation and abuse-report PDFs are built entirely in your browser (client-side) and are never uploaded to or stored on the server.

How long we keep your input

When you submit input, FalconEye:

  1. Calculates a SHA256 hash of the input
  2. Queries the relevant third-party data providers and assembles the result
  3. Stores the assembled result (NOT the raw input) in a local SQLite cache, keyed by the hash, for 24 hours
  4. Returns the result to your browser

After 24 hours the cached result expires and is overwritten on the next matching query. The original input is not stored beyond the moments needed to process it.

For uploaded files specifically: the file bytes are held in memory during parsing only, then explicitly cleared. The parsed text proceeds through the normal cache pipeline above.

The LLM-powered tabs (Email Header, Dork Generator, and Script Decoder) send your input to Anthropic's API. Anthropic's data handling policy is at anthropic.com/legal/privacy. At the time of writing, Anthropic does not use API input to train their models by default.

Cloudflare

FalconEye sits behind Cloudflare for caching, DDoS protection, and TLS. This means Cloudflare sees every request you make to falconeye.osintph.info, including your IP address and the URL path. Cloudflare may set its own cookies for security purposes (e.g., bot detection). See Cloudflare's privacy policy for details.

Stats page

The /stats/ path serves a GoAccess dashboard accessible only by the operator (gated by Cloudflare Access with operator-only authentication). General users do not have access to traffic stats. The data shown in the stats dashboard is aggregated from nginx access logs and contains no email content, no investigated indicators, and no LLM input/output.

Self-hosting

FalconEye is open source under AGPL-3.0. If you do not want to send your input to the live instance at falconeye.osintph.info, you can self-host your own copy from github.com/osintph/falconeye. Self-hosted instances do not communicate with this site or its operator.

Your rights

Because FalconEye does not maintain user accounts and stores only short-lived caches keyed by content hash (not by user identity), most data subject rights (access, deletion, correction) are not applicable in the traditional sense. If you believe your IP address or some input you submitted is being retained inappropriately, contact us using the address below and we will investigate.

If you are based in the Philippines, you have rights under the Data Privacy Act of 2012 (RA 10173). If you are based in the EU/EEA, you have rights under GDPR. If you are based in California, you have rights under CCPA.

Children

FalconEye is intended for use by cybersecurity professionals, OSINT investigators, journalists, and security researchers. It is not directed at children under 16. We do not knowingly collect data from children.

Changes to this policy

If we change how FalconEye handles data, we will update this policy and update the "Last updated" date at the top. For material changes (new third parties, new data categories), we will also make the modal display a "Policy updated" indicator for 14 days.
